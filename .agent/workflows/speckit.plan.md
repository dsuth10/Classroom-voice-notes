# /speckit.plan

Use this command to create a technical implementation plan for a feature defined in `specs/features/<feature-name>/spec.md`, based on the template at `specs/templates/plan-template.md`.

## Workflow

1.  **Analyze Specs**: Review `specs/features/<feature-name>/spec.md` and any related research.
2.  **Socratic Review**: Ask clarifying questions if anything is ambiguous.
3.  **Generate Plan**: Create `specs/features/<feature-name>/plan.md`.
4.  **Validate**: Ensure the plan adheres to the constitutional principles in `specs/constitution.md`.

## Template
The plan should follow the format of `specs/templates/plan-template.md`:
- Goal Description
- Technical Stack/Context
- Proposed Changes (Files/Components)
- Verification Plan
