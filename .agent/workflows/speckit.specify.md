# /speckit.specify

Use this command to create a `spec.md` file (or update an existing one) for a new feature inside the `specs/features/<feature-name>/` directory, based on the template at `specs/templates/spec-template.md`.

## Workflow

1.  **Understand Intent**: Clarify what the user wants to build.
2.  **Draft Specs**: Create `specs/features/<feature-name>/spec.md` including:
    - Executive Summary
    - User Stories (prioritised)
    - Functional Requirements
    - Technical Constraints (referencing the Constitution at `specs/constitution.md`)
3.  **Iterate**: Refine based on user feedback.
